# 确定性报告生成配套调整清单

目标：让主研究智能体产出完整、结构化、可追溯的 `research_result`，报告生成阶段只走确定性渲染，不再依赖 ReACT/LLM report agent。

## 调整项

- [x] `ResearchSection` 显式增加 `sources` 字段，保证章节级来源不会在模型校验时丢失。
- [x] `save_research_section` 保存章节时保留标准化后的 `sources`，同时继续同步项目级 `sources`。
- [x] `save_research_section` 在章节复用项目级来源时，自动把被证据链引用的来源回填到章节级 `sources`。
- [x] `ResearchResult` 增加全局 `synthesis` 字段，用于首页摘要、核心结论、跨章节洞察、建议和全局风险。
- [x] 从已保存章节和来源中确定性聚合 `fact_cards`、`insight_cards` 和 `synthesis`，避免报告层补写研究内容。
- [x] `generate_report` 改为直接调用确定性 `write_html_report`，不再优先调用 LLM report agent。
- [x] `build_research_agent` 不再构建独立 report agent，减少报告生成链路成本和不确定性。
- [x] 更新 `research_manager.md`，要求主研究智能体为确定性渲染提供章节 sources、表格/图表结构、风险说明和全局研究素材。
- [x] 运行快速验证：`compileall`、确定性渲染 smoke test。

## 验收记录

- [x] 代码编译通过。
  - 命令：`.venv/bin/python -m compileall app`
- [x] 使用构造的 `research_result` 能直接确定性生成 HTML。
  - `write_html_report` smoke test：`html_len=5910`，`sources=1`
- [x] 生成结果包含非空 `sources`。
  - `ResearchAgent.generate_report` smoke test：`html_len=5963`，`sources=1`
- [x] 生成结果不需要调用 LLM report agent。
  - `ResearchAgent(manager_agent=None, report_agent=object()).generate_report(...)` 通过，未访问 `report_agent`。
