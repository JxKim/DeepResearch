# 端到端测试记录

## 测试目标

验证 DeepResearch 后端主链路是否可以真实跑通：

1. 创建研究项目
2. 自动生成研究任务书和大纲
3. 获取大纲
4. 修改大纲
5. 获取修改后的大纲
6. 确认大纲
7. 生成研究结果和 HTML 报告
8. 获取最新报告

### 问题 1

- 发生步骤：步骤 2，轮询大纲生成任务
- 发生时间：2026-06-08 20:01:00
- 现象：创建项目成功后，后台大纲生成任务立即失败。
- 错误日志：`ImportError: Initializing ChatDeepSeek requires the langchain-deepseek package. Please install it with pip install langchain-deepseek`
- 初步判断：项目配置为 `deepseek` provider，但缺少 LangChain 对应 provider 包 `langchain-deepseek`。
- 修复方式：加入 `langchain-deepseek` 并同步虚拟环境。
- 修复涉及文件：`pyproject.toml`、`uv.lock`
- 复测结果：2026-06-08 20:03:14 复测不再出现 `ImportError`，该问题关闭；随后暴露新的 `TypeError`。
- 是否关闭：是

### 问题 2

- 发生步骤：步骤 2，轮询大纲生成任务
- 发生时间：2026-06-08 20:03:15
- 现象：依赖修复后，后台大纲生成任务仍失败。
- 错误日志：`TypeError: Object of type datetime is not JSON serializable`
- 初步判断：repository 返回的项目文档包含 `created_at/updated_at` 等 `datetime` 字段，`ResearchAgent._build_deepagents_input()` 直接 `json.dumps(payload)` 导致序列化失败。
- 修复方式：为 DeepAgents payload 的 `json.dumps` 增加 `default` fallback，`datetime/date` 转 ISO 字符串，其他非 JSON 原生对象转字符串。
- 修复涉及文件：`app/agents/research_agent.py`
- 复测结果：2026-06-08 20:09:23 HTTP 后台任务 `d133bb3c-24f6-460b-ab64-0f34b8ce56f9` 成功进入 `succeeded`。
- 是否关闭：是

### 问题 3

- 发生步骤：步骤 9，轮询报告生成任务
- 发生时间：2026-06-08 23:11:40
- 现象：报告生成任务返回 `succeeded`，但落库 `research_result` 中 `fact_cards`、`insight_cards`、章节正文和 `risks` 均为占位内容。
- 错误日志：任务日志显示 `完整研究结果已生成` 和 `研究和报告渲染完成`，但 MongoDB 复核显示 `research_result` 来自 `_build_placeholder_research_result()`。
- 初步判断：DeepAgents 实际执行了搜索和网页读取，但 `ResearchAgent._parse_research_result()` 没有正确解析最终研究 JSON，随后静默回退到占位研究结果；后台任务没有把“占位结果”识别为失败。
- 修复方式：待修复。需要禁止静默 fallback、解析 DeepAgents 最终 JSON/文件系统输出，并增加研究产物质量校验。
